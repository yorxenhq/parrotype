Privacy

The short version: everything happens on your computer.

Parrotype has no accounts, no telemetry, no analytics, no crash reporting. We don't know who you are, that you installed it, or how you use it — and we like it that way.

What stays on your machine

  • Your voice. Audio is captured, processed in memory and transcribed locally. It is not saved to disk and never leaves the computer.
  • Your text. Inserted into the active window and — if history is on — kept in a local file, capped at the last 50 dictations. You can copy, delete, clear all, or turn history off entirely in Settings.
  • Your settings and dictionary. Local files in %APPDATA%\Parrotype. Delete the folder and it's like we never met.

When it touches the network

Exactly two cases:

  1. Downloading the recognition model — once, on first run (or when you pick a different model). It's a public model file, fetched from Hugging Face; the request carries no personal data.
  2. An optional weekly version check — one anonymous request to GitHub asking what the latest release is. Nothing about you is sent. Off-switch is in Settings.

That's the complete list. The dictation core contains no network code at all — the repository is public, see for yourself:

grep -rniE "http|socket|urllib|requests" core/
